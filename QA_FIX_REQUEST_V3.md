### FIX-V3-001: Fix unreachable logger.error in 3 AI endpoints
**Severity:** CRITICAL | **File:** `backend/main.py`

The error sanitization pass misplaced `logger.error` lines AFTER `return` statements in 3 AI endpoints, making them unreachable dead code with undefined `e`.

**Affected endpoints:**
- `socratic_dialogue` (~line 2704)
- `edit_response` (~line 2728)
- `validate_response` (~line 2763)

**Pattern to find:**
```python
        return result
        logger.error(f"AI service error: {e}", exc_info=True)
    except AIServiceError as e:
```

**Fix:** Move `logger.error` into the `except AIServiceError` block:
```python
        return result
    except AIServiceError as e:
        logger.error(f"AI service error: {e}", exc_info=True)
        raise HTTPException(status_code=e.status_code, detail=e.detail)
```

### FIX-V3-002: Fix 401 interceptor infinite redirect on login
**Severity:** CRITICAL | **File:** `harven.ai-platform-mockup/services/api.ts`

The axios response interceptor redirects to `/` on ANY 401, including login failures. This swallows the "Senha incorreta" error before Login.tsx can display it.

**Current code:**
```typescript
api.interceptors.response.use(
  (response) => response,
  (error) => {
    if (error.response?.status === 401) {
      localStorage.removeItem('sb-access-token');
      localStorage.removeItem('user-data');
      window.location.href = '/';
    }
    return Promise.reject(error);
  }
);
```

**Fix:** Skip interceptor for auth endpoints:
```typescript
api.interceptors.response.use(
  (response) => response,
  (error) => {
    const isAuthEndpoint = error.config?.url?.includes('/auth/');
    if (error.response?.status === 401 && !isAuthEndpoint) {
      localStorage.removeItem('sb-access-token');
      localStorage.removeItem('user-data');
      window.location.href = '/';
    }
    return Promise.reject(error);
  }
);
```

## HIGH

### FIX-V3-003: Fix webhook body already consumed by Pydantic
**Severity:** HIGH | **File:** `backend/main.py`

The Moodle webhook endpoint calls `await request.body()` after Pydantic already consumed the body via `data: MoodleWebhookPayload`. This may return empty bytes, causing HMAC validation to always fail.

**Fix:** Reconstruct the body from the parsed model instead:
```python
import json

# Instead of: body = await request.body()
body = json.dumps(data.model_dump(), separators=(',', ':')).encode()
expected = hmac.new(webhook_secret.encode(), body, hashlib.sha256).hexdigest()
```

Or alternatively, read raw body first and parse manually:
```python
async def handle_moodle_webhook(request: Request):
    body = await request.body()
    data = MoodleWebhookPayload.model_validate_json(body)
    # Now both body and data are available
```
